Xcode多Target笔记

2014-12-14
项目实战

###由于之前印象看过使用多target来构建大量类似App这篇文章，知道有方法，所以一开始做的时候完全不考虑ipad版本，我只想着把iphone做出来，之后再考虑iPad的版本，因为我总是想着方法总是有的，我只管一个劲儿赶项目进度。只是这样好像没有比较快，我花了半天的时间把公共的部分抽出来，并且还遇到了一些问题。

###1.首先新建一个pad Target。

###2.双击原来的Target改名字，我是加了个_iPhone

###选择相应的Target,就会编译相应的包。

###3.这时候我就考虑两个Target公用的东西抽出来,我把原先的文件先Remove Reference.在Finder里整理好后，再拖入工程。我把公用的放入一个Common文件夹里，里面放了Lib_Common（放第三方库，网络层，数据层，一些Uitls等）,UI_Common（放一些基类，共用的xib,cell等）,Prefix,Common_Image.xcassets（共用的图片），Localizable.strings。然后再拖入工程中，拖的时候注意要勾选两个Target,

###整理后的工程结构看起来如图

###4.对了，由于我使用了CocoaPods管理第三方库，所以要修改下Podfile,增加一行link_with ‘YiRenMei_iPhone’,’YiRenMei_iPad’然后执行pod install. 这样新建的Target也能使用第三方库。

###5.开始编译，改动之后我必须保证原来的iPhone版本，不受影响，一堆错，

####由于我重新整理下目录导致了一些文件找不到的路径错误，还有其他的一些文件没有加到新Target里产生的错误。

####5.1 比如移动了 YiRenMei-Prefix.pch 文件，产生clang: error: no such file or directory:xxxx/YiRenMei-Prefix.pch
Command /Applications/Xcode.app/Contents/Developer/Toolchains/XcodeDefault.xctoolchain/usr/bin/clang failed with exit code 1

####5.2 info.plist也被我移动了 产生 Automatic provisioning profile selection unavailable: A bundle identifier is required for automatic provisioning profile selection. Either enter a bundle identifier in the Info.plist, or select a provisioning profile to use in the build settings.错误。

如还有其他错误，一样可以看看是否路径改变了。

另本地化Localizable.strings， 也注意勾选到 新Target里。

####5.3 ld: symbol(s) not found for architecture armv7 clang: error: linker command failed with exit code 1 (use -v to see invocation)

解决方法 原来是相应的framework没有加入到我新建的pad target中。

####5.4 objc_msgSend 方法报错 Too many arguments to function call,expected 0,have3. 这个真郁闷，幸好网上找了方法，Enable Strict Checking of objc_msgSend Calls 设成No.

###6.终于编译成功，总的来说，就是要注意各Target 里Build Phases里Complie Sources ,Copy Bundle Resources的文件，这也正是不同Target最终出来的东西不一样的原因，后续还要考虑到如何设计共用的ViewController，为ViewController瘦身，采用MVVM模式，更好的抽出phone ,pad版共用的部分，待更新。
